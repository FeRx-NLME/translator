# ferxtranslate v0.1 — Implementation Plan

Translate NONMEM, nlmixr2, and Monolix models into ferx `.ferx` format.
mrgsolve is v0.2 scope.

## Scope

| What to build | Approx. lines | Formats covered |
|---|---|---|
| `ferx_ir` + `emit_ferx()` + `rxui_to_ir()` | ~350 | NONMEM, nlmixr2, Monolix |
| mrgsolve block parser (v0.2) | ~400 | mrgsolve |

Strategy: `nonmem2rx` and `monolix2rx` (both on CRAN) do the hard parsing and
return a `rxode2` UI object. We write one adapter (`rxui_to_ir()`) that converts
that object to a ferx-native intermediate representation, then emit a `.ferx`
file. The nlmixr2 path is free on top of this because nlmixr2 model functions
ARE rxode2 UI objects.

## Step 2 — `ferx_ir` S3 class (Day 1, ~2 hours)

**File: `R/ir.R`**

The intermediate representation captures every concept expressible in a `.ferx`
file. All parsers produce this; the emitter consumes it. Names are stored
ferx-native (uppercase, underscored) from the moment of ingestion.

```r
new_ferx_ir <- function(
  source_format = NA_character_,   # "nonmem" | "nlmixr2" | "monolix" | "mrgsolve"
  source_file   = NA_character_,
  thetas        = list(),   # list of list(name, init, lower, upper)
  omegas        = list(),   # list of list(type, names, values)
  #                           type = "diagonal": names=c("ETA_CL"), values=0.07
  #                           type = "block":    names=c("ETA_CL","ETA_V"),
  #                                              values=c(v11, v21, v22)
  kappas        = list(),   # list of list(name, value)        -- IOV
  sigmas        = list(),   # list of list(name, value, scale) -- scale "sd"|"var"
  indiv_params  = list(),   # list of list(lhs, rhs)
  structural    = list(),   # list(type="pk_macro", pk_call, pk_args)
  #                           OR list(type="ode", obs_cmt, states)
  odes          = list(),   # list of list(state, rhs)  -- only if structural$type=="ode"
  diffusion     = list(),   # list of list(state, value)
  error_model   = list(),   # list of list(dv, type, params)
  #                           type = "proportional"|"additive"|"combined"
  scaling       = list(),   # list(obs_scale=NULL)
  fit_options   = list(),   # list(method, maxiter, covariance, iov_column)
  warnings      = character(),    # INFO/WARN/ERROR prefixed strings
  unsupported   = character()     # features detected but not translated
)
```

Also implement:
- `validate_ferx_ir(ir)` -- stops with informative message on invalid structure
- `print.ferx_ir(ir)` -- concise summary (sections present, warning count)

**Tests (`test-ir.R`):**
- Construct minimal valid IR, no error
- `validate_ferx_ir()` rejects missing `structural$type`
- `print.ferx_ir()` runs without error

## Step 3 — `emit_ferx()` (Day 2, ~3 hours)

**File: `R/emit_ferx.R`**

Takes a `ferx_ir`, returns a single character string ready to write as a `.ferx`
file. No file I/O here; that lives in `translate.R`.

**Output format:**

```
# Translated from {source_format}: {source_file}
# Warnings: {n} -- run result$warnings for details

[parameters]
  theta TVCL(0.134, 0.001, 10.0)
  omega ETA_CL ~ 0.07
  block_omega (ETA_CL, ETA_V) = [0.07, 0.02, 0.02]
  kappa KAPPA_CL ~ 0.04
  sigma PROP_ERR ~ 0.01 (sd)

[individual_parameters]
  CL = TVCL * exp(ETA_CL)

[structural_model]
  pk one_cpt_oral(cl=CL, v=V, ka=KA)

[odes]                              # only if structural$type == "ode"
  d/dt(depot) = -KA * depot

[error_model]
  DV ~ proportional(PROP_ERR)

[scaling]                           # only if obs_scale is not NULL
  obs_scale = 1000

[fit_options]
  method     = focei
  maxiter    = 500
  covariance = true
  iov_column = OCC                  # only if kappas non-empty
```

**Formatting rules:**
- Theta bounds always explicit: `(init, lower, upper)` -- user must review
- Omega/sigma always on variance scale in the file (ferx default), unless
  `scale == "sd"` in the IR in which case append ` (sd)`
- All names uppercase with underscores
- Unsupported features emitted as `# WARNING: <message>` at the point where
  the feature would have appeared, so the user sees it in context

**Tests (`test-emit.R`):**
- 1-cpt oral IR -> exact expected string (snapshot)
- 2-cpt oral with covariates -> exact expected string (snapshot)
- Block omega -> correct `block_omega` line
- ODE IR -> `[odes]` section present, `[structural_model]` has `ode(...)`
- IOV IR -> `kappa` line + `iov_column` in fit_options
- Unsupported feature in IR -> `# WARNING:` comment in output at correct position

Build and test section by section -- do not write the full emitter before
testing each section.

## Step 4 — `rxui_to_ir()` adapter (Day 3-4, ~5 hours)

**File: `R/rxui_to_ir.R`**

The rxode2 UI object (`rxUi` S3 class, environment-based) has this structure:

```
ui$iniDf    data.frame with columns:
              ntheta   -- integer, NA for omega rows
              neta1    -- integer, NA for theta rows; eta index
              neta2    -- integer; for off-diagonal neta1 != neta2
              name     -- character parameter name
              lower    -- numeric lower bound
              est      -- numeric initial value
              upper    -- numeric upper bound
              fix      -- logical
              err      -- character error type ("add","prop","add+prop") or NA
              condition-- "id" for IIV; occasion name for IOV

ui$lstExpr  list of R expressions (one per model line)
```

### 4a. Extract thetas

```r
theta_rows <- subset(ini, !is.na(ntheta) & is.na(err))
# -> list(name=row$name, init=row$est, lower=row$lower, upper=row$upper)
# If fix==TRUE: append WARNING "THETA {name} was FIXED -- treated as free in ferx"
```

### 4b. Extract omegas

```r
iiv_rows <- subset(ini, !is.na(neta1) & condition == "id")
# Diagonal: rows where neta1 == neta2
# Block: group consecutive eta indices with off-diagonal entries (neta1 != neta2)
# Each block: type="block", names=c(...), values=lower-triangular entries
```

### 4c. Extract kappas (IOV)

```r
iov_rows <- subset(ini, !is.na(neta1) & condition != "id")
# -> kappa entries; record condition value as iov_column in fit_options
# If any iov omega has off-diagonal: add WARNING, keep diagonal only
```

### 4d. Extract sigmas

```r
sigma_rows <- subset(ini, !is.na(err))
# err column gives error type; value in est is on SD scale in rxode2 UI
# -> scale = "sd" in ferx_ir sigma entry
```

### 4e. Parse model expressions

Walk `ui$lstExpr` and classify each expression:

```r
is_ddt(expr)        # call[[1]] == quote(`d/dt`)  -> odes entry
is_tilde(expr)      # call[[1]] == quote(`~`)     -> error_model
is_lincmt(expr)     # tilde with linCmt() on LHS  -> pk_macro path
is_assignment(expr) # call[[1]] %in% c("<-","=")  -> indiv_params
```

### 4f. linCmt() -> ferx pk macro

When `linCmt()` is detected, infer macro from parameters assigned in
`indiv_params`:

```r
params <- tolower(names of all indiv_param lhs values)

pk_macro <- if      ("ka" %in% p & "q2" %in% p & "q3" %in% p) "three_cpt_oral"   # warn
            else if ("ka" %in% p & "q"  %in% p)                "two_cpt_oral"
            else if ("ka" %in% p)                               "one_cpt_oral"
            else if ("q2" %in% p & "q3" %in% p)               "three_cpt_iv_bolus"
            else if ("q"  %in% p | "q2" %in% p)               "two_cpt_iv_bolus"
            else                                                "one_cpt_iv_bolus"
```

Check `indiv_params` for `f` (bioavailability) and `lagtime`/`alag` assignments
and include as optional pk args.

Note: `three_cpt_oral` does not exist in ferx -- emit WARNING and fall back to
the ODE path if the source has $DES, otherwise ERROR.

### 4g. Error model conversion

```r
# rxode2 syntax  ->  ferx type
# ~ prop(x)      ->  proportional
# ~ add(x)       ->  additive
# ~ add(x)+prop(y) -> combined   (check ferx supports this; warn if not)
```

The DV variable in ferx is always `DV`. The LHS of the tilde in rxode2 is
typically `cp` or a derived concentration -- discard it and use `DV`.

### 4g-ii. ODE auxiliary variable inlining

`$DES`-internal intermediates (e.g. `C2 = A(2)/V2`, `EFF = 1 - C2*EMAX/(EC50+C2)`) are
present in `ui$lstExpr` but must NOT appear in `[individual_parameters]` — they are
compartment-internal and undefined in ferx. If their names survive into ODE RHS strings,
the emitted `.ferx` file references undefined variables and will fail validation.

**Algorithm (implemented as Pass 2b in `rxui_to_ir.R`):**

After Pass 1 (d/dt extraction) and Pass 2 (assignment classification):

```r
# aux_vars  = upper-case names of all ODE state compartments
# sigma_upper = upper-case sigma names (EPS1 etc.)
# all_assigns = every lhs=..., rhs_expr=... seen in model block

# Build aux_map: intermediates that appear in the ODE block but are
# not states, not sigmas, and not already in indiv_params
aux_map <- list()
for (a in all_assigns) {
  if (a$lhs %in% aux_vars && !a$lhs %in% state_upper && !a$lhs %in% sigma_upper)
    aux_map[[a$lhs]] <- normalise_expr(a$rhs_expr, name_map)
}

# Recursively substitute aux_map into each ODE RHS expression
odes <- lapply(odes, function(o) {
  inlined <- .inline_aux_vars(o$rhs_expr, aux_map)
  list(state = o$state, rhs = deparse(inlined))
})
```

`.inline_aux_vars()` is a recursive expression walker (depth-capped at 30) that
replaces symbol references with their defining expressions. It must store
`rhs_expr_norm` on each ode entry during Pass 1 so the expression is available here.

### 4h. Naming normalisation

Apply on ingestion (in `rxui_to_ir`), not in emit:
- `eta.cl` -> `ETA_CL`
- `err.prop` -> `PROP_ERR`
- `tvcl` -> `TVCL`
- Rule: `toupper(gsub(".", "_", name, fixed=TRUE))`

**Tests (`test-rxui_to_ir.R`):**

Build rxode2 UI objects inline using `rxode2::rxode2()` functions -- no files needed:

```r
f_1cpt <- function() {
  ini({ tvcl <- 0.134; tvv <- 8.1; tvka <- 1.0
        eta.cl ~ 0.07; eta.v ~ 0.02
        err.prop ~ 0.01 })
  model({ cl <- tvcl*exp(eta.cl); v <- tvv*exp(eta.v)
          ka <- tvka
          linCmt() ~ prop(err.prop) })
}
```

Tests:
- 1-cpt oral linCmt() -> `structural$pk_call == "one_cpt_oral"`
- 2-cpt oral linCmt() with q, v2 -> `"two_cpt_oral"`
- ODE model with d/dt() -> `structural$type == "ode"`, odes populated
- Block omega (off-diagonal in iniDf) -> `omegas[[1]]$type == "block"`
- IOV condition != "id" -> kappas populated, iov_column set
- `fix=TRUE` theta -> warning added to ir$warnings
- ETA naming: `eta.cl` -> `ETA_CL` in indiv_params rhs

## Step 5 — `to_ferx()` and format shortcuts (Day 5, ~2 hours)

**File: `R/translate.R`**

```r
to_ferx <- function(source,
                    format    = c("nonmem", "nlmixr2", "monolix"),
                    output    = NULL,
                    overwrite = FALSE) {
  format <- match.arg(format)
  rxui <- switch(format,
    nonmem  = nonmem2rx::nonmem2rx(source),
    nlmixr2 = rxode2::rxode2(source),
    monolix = {
      if (!requireNamespace("monolix2rx", quietly = TRUE))
        cli::cli_abort(c(
          "monolix2rx is required for Monolix translation.",
          i = "Install it with: {.code install.packages('monolix2rx')}"
        ))
      monolix2rx::monolix2rx(source)
    }
  )
  ir     <- rxui_to_ir(rxui, source_format = format,
                       source_file = if (is.character(source)) source else NA)
  text   <- emit_ferx(ir)
  result <- new_ferx_translate_result(text, ir)
  if (!is.null(output)) write_ferx(result, output, overwrite)
  invisible(result)
}

nm_to_ferx      <- function(ctl_file, output = NULL, ...) to_ferx(ctl_file,  "nonmem",  output, ...)
nlmixr2_to_ferx <- function(model_fn, output = NULL, ...) to_ferx(model_fn,  "nlmixr2", output, ...)
mlx_to_ferx     <- function(mlxtran,  output = NULL, ...) to_ferx(mlxtran,   "monolix", output, ...)
```

**`ferx_translate_result` S3 class:**

Fields:
- `$ferx_text`   -- the .ferx file as a single character string
- `$warnings`    -- character vector, INFO/WARN/ERROR prefixed
- `$unsupported` -- features detected in source but not emitted

`print.ferx_translate_result`:
1. Header: "Translated from {format}: {file}"
2. The full `.ferx` text (cat, not print)
3. If warnings: yellow section "Translation warnings:" listing each
4. If unsupported: red section "Unsupported features (manual fix required):"

`write_ferx(result, path, overwrite)`:
- If file exists and `overwrite=FALSE` -> `cli::cli_abort()` with clear message
- Writes `result$ferx_text` to path
- Returns `result` invisibly for pipe use

## Step 6 — Warning system

Three levels. All stored in `ir$warnings` and surfaced via `print.ferx_translate_result`.

```
INFO  | THETA TVCL was FIXED in source -- treated as free in ferx
WARN  | Theta bounds not found for THETA2 -- set to (init, 0, Inf), please review
WARN  | $ERROR block was complex -- classified as proportional, verify against original
ERROR | MIXTURE model detected -- not supported in ferx, removed from output
ERROR | Multiple DVIDs detected -- ferx supports one DV; secondary endpoint removed
```

`cli::cli_warn()` emitted at translation time so the user sees warnings
immediately. Stored in `ir$warnings` for programmatic inspection
(`result$warnings`).

**amp.sim reference models (`LeidenAdvancedPKPD/amp.sim`):** amp.sim is a simulation
benchmarking package that ships NONMEM reference control streams with known true
parameter values. These are NOT DDMoRe models; they test our adapter against realistic
model patterns (minimal bounds, ODE-only, compartment initial conditions via `A_0()`)
that the simpler test models do not exercise.

Note: amp.sim `$DES`-internal intermediates (`C2`, `EFF`) exercise the aux-var
inlining path (Step 4g-ii). The initial condition `A_0(4)=BL` is silently dropped
by nonmem2rx and is outside our control; a WARN-level advisory is appropriate.

## Step 10 — Numerical concordance tests (v0.1 gate, Day 8) [COMPLETE]

Snapshot tests verify that text output looks right. Concordance tests verify that
the translated `.ferx` model actually fits data and reproduces NONMEM parameter
estimates within tolerance.

### Status: DONE

All concordance tests implemented and passing locally (258 total tests, 0 failures).
Two bugs found and fixed during implementation:

**Bug 1 (fixed): Fixed-effect PK param passthrough** (`rxui_to_ir.R`)
- When a PK param is defined as `V = THETA(3)` (no ETA), nonmem2rx does not include
  it in `indiv_params`. The pk macro arg lookup then misses `v=V`, producing IPRED=0.
- Fix: after ODE/linCmt detection, scan thetas for standard PK names not already in
  `indiv_params` and add passthrough entries (`V = V`). Affects any model with a
  fixed-effect volume, clearance, etc.

**Bug 2 (fixed): S2=V compartment scaling** (`utils.R`, `translate.R`, `rxui_to_ir.R`, `emit_ferx.R`)
- NONMEM `S2 = V` means "divide compartment 2 amount by V before comparing to DV
  (which is a concentration)". nonmem2rx drops this from `lstExpr`, so the ODE model
  was predicting amounts and comparing them to concentration data -- producing IPRED >> DV.
- Fix: `.extract_nm_scaling()` in `utils.R` parses the raw `.ctl`/`.mod` file for
  `Sn = varname` assignments in `$PK`. `to_ferx()` passes the result as `scaling_hint`
  to `rxui_to_ir()`, which emits `[scaling] obs_scale = V` in the output.

### Concordance test suite

`tests/testthat/test-concordance.R` -- gated with `skip_if_not_installed("ferx")` and
`skip_on_ci()`. Datasets in `inst/testdata/` simulated via `ferx_simulate()` at known
theta values. See `data-raw/generate_concordance_data.R` for reproducible generation.

| Test | Model | Path | True params | Tolerance | Actual recovery |
|---|---|---|---|---|---|
| linCmt 1-cpt oral: TVCL/TVV | `1cpt_oral.ctl` | linCmt | TVCL=0.134, TVV=8.1 | 15% | <5% |
| linCmt 2-cpt IV: 4 thetas | `2cpt_iv.ctl` | linCmt | CL=5, V1=20, Q=8, V2=60 | 10% | <2% |
| linCmt 2-cpt IV: omegas | `2cpt_iv.ctl` | linCmt | om_CL=0.10, om_V1=0.10, om_Q=0.08 | 10% | <1% |
| amp.sim linCmt: KA/CL/V | `pk_1cmt_oral_ampsim.ctl` | linCmt | NONMEM FOCEI reference | 10% | KA -2%, CL -3%, V -0.3% |
| ODE 1-cpt oral with S2=V | `pk_1cmt_oral.mod` | ODE | KA=0.1, CL=2.0, V=1.0 | 15% | (run locally) |

Design decisions:
- 1-cpt oral omega tests removed: omega_V=0.02 too small to estimate from 100 subjects
  (39% error). Only 2-cpt IV omegas (0.05-0.10) tested, where recovery is <1%.
- TVCL/TVV only for 1-cpt oral: TVKA (omega=0.4) has high variance; excluded from strict tolerance.
- amp.sim test uses `pk_1cmt_oral_ampsim.ctl` (ADVAN2/linCmt), not `pk_1cmt_oral.mod` (ADVAN6/ODE),
  because the linCmt path is tested separately and the amp.sim external reference is the key value.
- `NM.theoph.02B.csv` is not bundled in amp.sim; data simulated from reference estimates via ferx_simulate.

### Why this matters

The DDMoRe project (EU drug-disease model resource, 2012-2016) demonstrated that
cross-tool translation is error-prone even for simple PK models: factor-of-2
discrepancies in estimated parameters were common between tools claiming to support
the same model. Numerical concordance tests are the only way to catch silent
semantics errors -- wrong ODE RHS sign, swapped parameter, wrong sigma scale -- that
snapshot tests will never detect.

## What cannot be parsed -- ferx feature roadmap

Features detected during translation that ferx does not yet support.
Each is emitted as a `# WARNING:` comment in the output file.

### Translates with loss (WARN) -- result is usable but imperfect

| Source feature | What we emit | Ferx work needed |
|---|---|---|
| `THETA (FIX)` | Free theta + WARNING | `fix` keyword on theta declaration |
| FO estimation | `method = foce` + WARNING | none (FO is interior to FOCE) |
| BAYES / NUTS estimation | `method = saem` + WARNING | Full Bayesian / HMC support |
| No theta upper bound in source | `(init, 0, Inf)` + WARNING | none -- user must supply |

### Untranslatable -- removed from output (ERROR)

| Source feature | Ferx feature needed |
|---|---|
| Multiple DVIDs / endpoints | Multi-endpoint / DVID support |
| PD structural models (Emax, indirect response) | PD model primitives |
| `MIXTURE` models | Mixture model support |
| `R1`/`D1` model-defined infusion rate/duration | Rate parameter in pk macro |
| `ALAG2`, `ALAG3` (lag time on non-absorption cmts) | Per-compartment lag time |
| `F2`, `F3` (bioavailability on non-absorption cmts) | Per-compartment bioavailability |
| `S2`, `S3` (per-compartment scaling) | Per-compartment obs_scale |
| `$SIGMA BLOCK` (correlated residuals) | Correlated residual errors |
| Categorical / count / TTE endpoints | Non-continuous likelihoods |
| NONMEM `$CONTR`, `$INFN`, `VERBATIM` blocks | Not applicable |
| ADVAN5 (general linear, >3 cpts) -- analytical only | Forced to ODE path (usually fine) |
